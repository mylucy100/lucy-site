OpenClaw 作为开源 AI Agent 平台，近期发布了 **2026.3.1** 版本。这个版本包含了超过 200 个 commit，重点提升了 Agent 推理能力、容器健康检查、Android 节点功能以及消息平台集成。本文将深入解读这次更新的核心亮点。

## AI Agent 推理增强

### 自适应思考级别

最引人注目的变化是为 **Anthropic Claude 4.6 模型**（包括 Bedrock 变体）设置了 "adaptive" 作为默认思考级别，而其他推理模型默认使用 "low"。

这个调整让 Agent 能够根据任务复杂度动态调整认知投入：
- **简单查询**：降低延迟，快速响应
- **复杂分析**：深度思考，确保质量

开发者仍然可以通过显式配置来精细调优生产环境的表现。

### OpenAI 流式传输升级

OpenAI 流式传输转向 **WebSocket 优先模型**：
- 共享连接线路
- 服务端上下文压缩
- 减少长会话中的负载膨胀

这对需要长时间对话的 Agent 场景特别有价值。

### 子代理运行时事件

临时完成的交接现在结构化为 `task_completion` 事件，确保直接路径和队列路径的一致渲染，非常适合在 Gateway 或 CLI 中调试分布式 Agent 集群。

## 容器与 Gateway 增强

### 内置健康检查端点

Gateway/容器探针现在包含内置 HTTP 端点：

```
/health
/healthz
/ready
/readyz
```

这些端点专为 Docker 和 Kubernetes 健康检查设计，在路径冲突时会回退到现有处理程序。这对在云原生环境中扩展 AI Agent 至关重要。

### 轻量级 Cron 启动模式

Cron 任务新增轻量级引导模式：
- CLI 参数：`--light-context`
- 配置项：`agents.*.heartbeat.lightContext`

该模式跳过重文件注入，仅保留 HEARTBEAT.md 核心内容，显著减少启动开销，让定时 Agent 在资源受限的节点上更快启动。

## Android 节点扩展

Android 节点在平台平等方面领先，新增了多项功能：

| 功能 | 说明 |
|------|------|
| `camera.list` | 相机列表 |
| `device.permissions` | 设备权限 |
| `device.health` | 设备健康状态 |
| `notifications.actions` | 通知操作（打开/关闭/回复） |

新增的传感器包括：
- 系统、通知、照片最新
- 联系人搜索/添加
- 日历事件添加
- 运动活动/计步器

这些功能让 Agent 能够在移动设备上原生运行，无需抖动的轮询机制。

## 消息平台优化

### Discord 线程生命周期

Discord 线程从固定 TTL 改为基于不活跃的生命周期管理：
- 默认 `idleHours: 24`
- 可选 `maxAgeHours`
- 新增 `/session idle` 和 `/session max-age` 命令

### Telegram DM 主题

Telegram DM 主题支持：
- 每个 DM 的独立配置（白名单、dmPolicy、技能、系统提示）
- 作为独立会话路由
- 主题感知认证和防抖

### 飞书大规模修复

飞书集成获得了大量改进：
- Docx 表格创建/写入
- 图片/文件上传
- 反应处理
- 聊天工具
- 权限授予
- 群组会话范围

### Slack 改进

Slack 改进了公告路由：
- 基于会话的账户查找
- 标准化用户令牌获取
- 多账户操作支持

## 安全强化

本次更新的安全修复值得关注：

### 飞书 Webhook 限流

飞书 Webhook 入站添加了有界速率限制和修剪，防止密钥轮换攻击导致的内存耗尽。

### WebSocket 连接管理

Gateway WebSocket 在握手后关闭未授权洪水连接，并采样日志避免日志垃圾信息。

### Android 通知去重

Android 通知唤醒增加了摘要去重和压缩审计丢弃泄漏消息。

## 更新总结

| 功能领域 | 关键新增 | 影响 |
|---------|---------|------|
| AI Agent | 自适应 Claude 4.6 思考、WS 流式、子代理事件 | 更智能、高效的推理链 |
| Android 节点 | 传感器、通知、相机工具 | 原生移动 Agent 操作 |
| 飞书集成 | Docx 表格、反应、群组范围 | 企业级消息就绪 |
| Gateway/容器 | 健康端点、轻量 Cron 引导 | Kubernetes/Docker 扩展 |
| 安全强化 | 飞书 Webhook 限流、WS 洪水关闭 | 生产环境韧性 |
