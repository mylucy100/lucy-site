AI 开发游戏的玩法，又升级了。

今天在 GitHub Trending 上发现了一个有趣的项目：**Claude Code Game Studios**，直接斩获 **13,000+ Stars**。

## 这是什么？

简单来说，它把一个 Claude Code 变成了**一整个游戏开发工作室**。

有 49 个专业 AI 员工，72 项技能，还有完整的组织架构。创始人说：

> "Building a game solo with AI is powerful — but a single chat session has no structure."

（用 AI 单独做游戏很强大，但单个对话没有结构。）

## 工作室架构

### 人员配置（49 个 Agent）

| 部门 | 人数 | 职责 |
|------|------|------|
| **Directors** | - | 守护项目愿景 |
| **Department Leads** | - | 负责各自领域 |
| **Specialists** | - | 执行具体工作 |

### 技能配置（72 项）

从 `/start` 到 `/story-done`，覆盖游戏开发的全流程：

- `/design-system` - 设计系统
- `/create-epics` - 创建史诗任务
- `/create-stories` - 编写故事
- `/dev-story` - 开发故事
- 等等......

### 质量门禁

这个项目最让我感兴趣的，是它引入了**真实工作室的质量控制**：

- **Design Review**：设计评审
- **QA Pass**：测试通过
- **Code Review**：代码审查
- **Escalation**：升级机制

## 能做什么？

### 开发流程示例

```
你：/start 新项目
    ↓
Director（项目总监）确认方向
    ↓
Design Lead 设计系统
    ↓
Programming Lead 分配任务
    ↓
N 位 Specialist 各自开发
    ↓
QA 测试
    ↓
Review 最终交付
```

### 解决什么问题？

1. **不会再有 magic numbers** - 有人帮你盯着
2. **不会跳过设计文档** - 必须先写 GDD
3. **不会写出 spaghetti code** - 代码规范强制执行
4. **不会再漏掉测试** - QA 是必须的

## 快速体验

```bash
git clone https://github.com/Donchitos/Claude-Code-Game-Studios.git
```

然后用 Claude Code 打开，按照 `/start` 提示操作即可。

## 我的思考

### 亮点

1. **结构化**：终于把 AI 开发从"聊天"变成了"工作流"
2. **职责明确**：每个 Agent 有明确的角色和权限
3. **可扩展**：Slash commands 设计很优雅

### 可能的局限

1. 需要熟悉他们的工作流命令
2. Agent 之间的协调可能需要调试
3. 目前专注于游戏开发，通用性有限

### 更大胆的想象

如果这套架构移植到**普通软件开发**会怎样？

- 49 个 Agent = 前端 + 后端 + 测试 + DevOps + 产品经理
- 72 项技能 = 完整软件工程流程
- 质量门禁 = 企业级开发标准

或许这才是 AI 软件开发的未来？

Claude Code Game Studios 展示了一种可能性：**把 AI 从"助手"变成"团队"**。

虽然目前主要面向游戏开发，但它验证的理念——多 Agent 协作 + 明确职责 + 质量门禁——可能代表了 AI 软件开发的下一个方向。
