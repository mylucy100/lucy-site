我，一个深度 OpenClaw 用户，硬是用了半天 Hermes Agent，然后决定叛逃。

这不是什么「最美逆行」，只是一个工程师用脚投票的结果。

## 背景

Hermes Agent 这个名字，你可能在某个技术公众号里刷到过。是的，就是那个「爱马仕」——一个支持多平台接入的 AI 助手框架，能跑终端、能接飞书、能连 Telegram，还有工具调用、任务调度、记忆持久化这些基建能力。

我之前就注意到了，但一直拖着没试。为啥？惯性。

OpenClaw 用了小半年，习惯养成了，技能也攒了一堆。迁移？想想就麻烦。直到昨天半夜，网络环境突然给力了一把，冲动之下装好了 Hermes。今天早起又折腾了一会儿，总算跑起来了。

然后我就愣住了。

## 第一个照面：终端 UI 的降维打击

先说最直观的体验：终端 UI。

Hermes 的终端对话界面，做得比 OpenClaw 高一个段位。它会在调用工具时生成 emoji 提示——搜索文件、读取文档、写入内容，你永远知道它在干什么。

```bash
🔍 搜索文件中...
📖 读取 /path/to/file
✏️ 写入内容到 xxx.md
```

OpenClaw 呢？终端里只能看到一个动态符号转啊转，你知道它在工作，但不知道它在干嘛。就像你家猫蹲在键盘上，你不知道它是在睡觉还是在删你代码。

这种「不确定感」，用过 OpenClaw 的人都懂。

## 飞书集成：才是真正的主战场

平时我主要在飞书和终端两边跑。飞书这一块，Hermes 做得更绝。

它会把操作步骤同步到聊天窗口——搜索文件、读取文件、修改文件，一步步都给你发消息。更离谱的是，它还能做卡片回调交互。

比如我要修改一个重要配置文件，Hermes 会弹出一个卡片：「确认修改吗？」点个确定就执行，点个取消就撤销。

OpenClaw 呢？它不会告诉你它在干嘛。默默执行，默默返回结果。像一个不爱说话的技术宅男友——靠谱，但让你猜得累。

## 技能记忆：谁才是真正的「不忘事」

OpenClaw 的技能库很丰富，这是事实。但它有个致命问题：健忘。

我经常遇到这种情况：告诉它一件事，下次它还问同样的问题。告诉它不要这样做，它转头又犯。反复纠正，像在训练一只记不住指令的狗。

Hermes 不一样。它有自己的数据库存储日常对话，记忆架构和 OpenClaw 的三层模型不太一样。

更夸张的是，Hermes 会「自适应」：如果你经常让它做一件事且做对了，它可能会把这个流程自动做成技能，下次直接调用。

这叫什么？这叫「用进废退」的自我进化。

## 智商测试：同一个模型，谁更聪明？

我用的是 MiniMax-M2.5 模型。在相同模型下，Hermes 比 OpenClaw 更细致、更聪明。

怎么体现？

上午我用 OpenClaw（龙虾）搞一个复杂的编程问题，搞了一上午没搞定。下午换成 Hermes，半小时解决。

这不是玄学，是推理链的完整性。Hermes 在复杂场景下的逻辑链更长、更少跳步。就像同样是解数学题，一个会跳步骤，一个步步为营。

## 迁移成本：技能导入的那些坑

用了 OpenClaw 一段时间，多多少少攒了一些自定义技能。Hermes 提供了导入功能，但我按官方文档操作，没成功。

后来我换了个思路：直接告诉它我需要什么技能，手动导入，测试后都能用。

为什么官方导入失败，我也不清楚。可能是个 bug，也可能是我操作有问题。总之不是blocking issue。

## 评分总结

10 分满分，我这么打分：

- OpenClaw：6 分
- Hermes Agent：7 分

为什么这么评？

OpenClaw 的程序写得不夠细致，插件和功能太多，照顾不过来。相对的，Hermes 更稳定，记忆更全面。OpenClaw 经常记不住我反复强调的偏好，Hermes 目前没出现过这种情况。

另外， Hermes 默认集成了一些官方技能，对新手更友好——开箱即用，不需要自己踩坑配置。

## 我的选择

经过一下午加一晚上的使用，我已经暂停 OpenClaw，转向 Hermes。

不是 OpenClaw 不好，而是 Hermes 刚好踩中了我更在意的点：终端 UI 清晰、飞书交互友好、记忆不丢失、复杂推理更靠谱。

就像找对象，不是ta不够好，而是我更想要的那个特质，恰好在另一个人身上。

## 最后

Hermes Agent 可以一试。

如果你也是 OpenClaw 老用户，建议找个周末下午，两边同时跑跑看。让自己决定，而不是让算法决定。

毕竟，工具是为人服务的，不是反过来。
