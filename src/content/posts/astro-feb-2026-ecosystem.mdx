Astro 的 2026 年开局可以用"狂飙"来形容。

## State of JS 2025：Astro 全面领跑

2025 年的 State of JS 调查结果出来了，Astro 的成绩单相当亮眼：

- **使用量 #2**（连续两年保持）
- **关注度 #1**（连续四年！）
- **满意度 #1**（连续三年）
- **正面评价 #1**（连续三年）

HTTP Archive 的数据也印证了这一点：Astro 在移动端占比 11.8%，桌面端 14.6%。这可不是小数目——意味着每 10 个网站里就有一个是 Astro 构建的。

## Astro 6 Beta 持续迭代

截至 2 月底，Astro 6 已经发布了 17 个 beta 版本。更重要的是，[升级指南草案](https://v6.docs.astro.build/en/guides/upgrade-to/v6/)已经可以查阅了。

如果你还没试过 Astro 6，现在是个好时机。主要变化包括：
- 开发服务器与生产环境统一运行时
- 原生 CSP 支持
- Live Content Collections 稳定版

详见我之前写的 [Astro 6 正式版发布](/posts/astro-6-stable-release)。

## AI 相关工具扎堆出现

这个月最让我兴奋的是 AI 集成工具的爆发。

### Frontman Astro

[Frontman Astro](https://npmx.dev/package/@frontman-ai/astro) 是一个 AI 集成，让你可以直接在浏览器里编辑前端。想象一下：部署后需要改个文案或调整布局，不用打开代码编辑器，直接在网页上点击修改就行。

对于非技术背景的内容运营来说，这是真正的解放。

### Agent CMS

[Agent CMS](https://npmx.dev/package/@agentcms/agentcms) 是专门为 Astro 6 + Cloudflare 设计的"AI-agent-first"无头 CMS。它的设计理念是：让 AI 智能体能直接操作内容，而不是通过人类间接操作。

这和 OpenClaw 的 Agent Skills 理念很像——AI 不只是生成代码，还要能管理内容、维护网站。

### Astro Markdown for Agents

[Astro Markdown for Agents](https://npmx.dev/package/@puralex/astro-markdown-for-agents) 自动为 AI 智能体提供 Markdown 内容——即使部署在 Cloudflare 的免费计划上也能工作。

这意味着 AI 可以直接读取你的网站内容，用于搜索、摘要或其他智能处理。

## SEO 和性能工具升级

### Astro Flyweb

[Astro Flyweb](https://npmx.dev/package/astro-flyweb) 自动生成 `/.well-known/flyweb.json`，让 AI 智能体能发现你网站的结构化内容。

这和 [Astro Skills](https://npmx.dev/package/astro-skills) 类似——都遵循"网站是 AI 的数据源"这个理念。如果你的网站想被 AI 搜索引擎更好地索引，这是个值得关注的工具。

### Subfont

[Subfont](https://npmx.dev/package/@forthgoing/subfont) 通过字体子集化显著减小字体文件大小。字体加载是很多网站的性能瓶颈，这个工具能在构建时优化，让加载速度更快。

### Astro ID Minifier

[Astro ID Minifier](https://npmx.dev/package/astro-id-minifier) 压缩 Astro 的自定义数据属性，减少构建体积。对于大型网站来说，每个字节都重要。

## 国际化工具百花齐放

这个月出现了不少于 5 个新的 i18n 工具：

- [Mannisto Astro i18n](https://npmx.dev/package/@mannisto/astro-i18n) — 灵活的替代方案
- [Arisakit Astro i18n](https://npmx.dev/package/@ariaskit/astro-i18n) — 类型安全的国际化
- [Otrodigital i18n Next](https://npmx.dev/package/@otrodigital/astro-i18n-next) — 静态站点的完整方案
- [Astro Intl](https://npmx.dev/package/astro-intl) — 简洁的类型安全系统

如果你的网站需要多语言支持，现在选哪个都行——关键是选一个并坚持下去。

## Starlight 文档生态也在扩张

Starlight（Astro 的文档框架）新增了几个实用插件：

- [Starlight Announcement](https://npmx.dev/package/starlight-announcement) — 自定义公告栏
- [Starlight Telescope](https://npmx.dev/package/starlight-telescope) — 模糊搜索 + 键盘导航

很多知名项目的文档都在用 Starlight，比如 GitHub 的 AW 文档、Zulip 帮助文档等。

## 我的观察

这个月 Astro 生态最明显的趋势是：**AI 工具正在改变我们维护网站的方式**。

从 Frontman 的"浏览器直接编辑"到 Agent CMS 的"AI-first 内容管理"，整个生态系统都在围绕 AI 智能体重新设计工作流。

如果你是 Astro 开发者，现在有三个方向值得关注：

1. **升级到 Astro 6** — 开发体验和安全都有质的提升
2. **探索 AI 集成** — 看看 Frontman 和 Agent CMS 能如何简化你的工作
3. **优化 SEO** — Flyweb 和 Markdown for Agents 让你的网站更容易被 AI 发现

Astro 不只是静态网站生成器了，它是 AI 智能体时代的内容平台基础。
