OpenClaw 在 2026 年 4 月迎来了一个重磅更新。

从 2026.3.13 到 2026.4.2，短短半个月发布了 **10 个版本**，开发节奏非常快。

这次更新带来了很多期待已久的功能：**QQ Bot 正式支持**、**后台任务控制面板**、**Slack 原生审批**……

让我们深入解读这次更新的核心内容。

## 一、全新频道插件：QQ Bot 正式登场

这是很多中国用户期待的功能。

QQ Bot 现已作为**内置频道插件**加入 OpenClaw，支持：

- **多账号配置** — 可以同时管理多个机器人
- **SecretRef 凭证管理** — 安全存储敏感信息
- **斜杠命令** — `/help`、`/status` 等命令支持
- **提醒功能** — 设置定时提醒
- **媒体收发** — 支持图片、文件等媒体内容

如果你在国内使用 OpenClaw，QQ Bot 让你的 AI 助手能直接接入 QQ 群和好友聊天，实用性大幅提升。

相关配置文档：[OpenClaw 官方文档](https://docs.openclaw.ai)

## 二、后台任务控制面板

这是一个架构层面的重大改进。

以前，ACP（Agent Control Protocol）、子智能体、cron 定时任务、后台 CLI 执行各自独立管理，比较分散。

现在统一为一个 **SQLite 后台任务控制面板**：

- **统一生命周期管理** — 所有后台任务在一个地方查看和控制
- **审计与维护** — 任务执行历史、状态可见性
- **自动清理** — 失效任务自动清理，避免资源浪费
- **丢失恢复** — 异常中断的任务可以恢复

新增了 `openclaw flows` 命令：

```bash
openclaw flows list   # 列出所有任务流
openclaw flows show   # 显示详情
openclaw flows cancel # 取消任务
```

这让后台任务的监控和调试变得更加直观。

## 三、Slack 原生审批路由

执行敏感命令时，需要人工审批确认。

以前在 Slack 通道，审批请求会跳转到 Web UI 或终端，体验不够连贯。

现在 **审批请求可以直接在 Slack 内完成**：

- 审批提示留在 Slack 通道
- 授权用户可以直接在 Slack 批准或拒绝
- 无需切换界面

这对使用 Slack 作为主要沟通渠道的团队来说，大大提升了工作效率。

## 四、WhatsApp 表情回应

一个有趣但实用的小功能。

AI 助手现在可以**用表情回应 WhatsApp 消息**。

比如收到一张照片，助手可以用 ❤️ 表示喜欢，而不是非要写一条回复文字。

这让对话更像真人交流，更自然。

## 五、LINE 媒体发送增强

LINE 通道现在支持：

- **图片发送**
- **视频发送** — 支持预览和追踪
- **音频发送**

之前 LINE 只支持图片，现在媒体类型更完整了。

## 六、Matrix 历史上下文

Matrix（一种去中心化聊天协议）支持现在更强大：

- **历史上下文** — 可以配置读取多少历史消息作为上下文
- **代理支持** — Matrix 流量可以通过 HTTP(S) 代理
- **流式回复** — 部分回复会更新同一条消息，而不是发送多条
- **DM 线程控制** — 私聊中可以控制线程行为

这让 Matrix 用户的使用体验更接近主流聊天平台。

## 七、安全加强：多项关键改进

这次更新在安全方面做了很多加强。

### 1. 危险代码扫描默认失败

安装插件或技能时，如果扫描发现危险代码：

- **之前**：可能仍然成功安装（需要手动检查）
- **现在**：默认拒绝安装，必须显式使用 `--dangerously-force-unsafe-install` 才能继续

这是一个「**失败即安全**」的设计理念。

### 2. 节点命令需配对批准

设备配对后，节点命令不会立即生效。

需要**显式批准**才能使用，防止未授权访问。

### 3. ACP 安全语义审批

ACP（Agent Control Protocol）的危险工具审批从「名称覆盖」改为「语义审批类」：

- 只读操作可以自动批准
- 可能执行代码或控制系统的工具必须显式批准

这是更智能的安全策略。

### 4. Nostr DM 签名验证

接收 Nostr 私聊消息时，会验证事件签名：

- 伪造的 DM 事件无法创建配对请求
- 无法触发回复

防止伪造消息攻击。

## 八、其他值得关注的改进

### MCP 远程 HTTP/SSE 支持

MCP（Model Context Protocol）服务器现在支持 HTTP/SSE 连接：

- 可以连接远程 MCP 服务器
- 支持认证头
- 凭证配置更安全

### OpenAI Responses 优化

- 支持配置 verbosity（详细程度）
- 在 `/status` 中显示

### TTS 诊断增强

语音合成服务增加了结构化诊断和回退尝试分析，更容易排查问题。

### 上下文修剪修复

之前估算中文和日文字符（特别是 Extension B 字符）时有偏差，导致上下文修剪不准确。现已修复。

## 小结：更安全、更完整、更可控

OpenClaw 2026.4 的核心变化可以用三个词概括：

1. **更安全** — 危险代码扫描、节点命令控制、ACP 语义审批
2. **更完整** — QQ Bot、LINE 媒体、WhatsApp 表情、Matrix 历史
3. **更可控** — 后台任务控制面板、flows 命令、Slack 原生审批

对于国内用户，QQ Bot 的加入是最大亮点。

对于企业用户，安全加强和 Slack 审批路由提升了可管理性。

对于所有用户，后台任务控制面板让复杂的自动化任务更容易监控。
